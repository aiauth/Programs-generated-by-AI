任务目标
请为我从零构建一个可直接在 DevEco Studio 中运行的 HarmonyOS 6.0 密码管理器应用。

应用名称：安盾密码 (SafeArmor)

核心功能简述：支持账号密码的分类管理、高强度 AES-256-GCM 本地加密存储、集成生物识别（指纹/人脸）解锁、自动生成强密码、以及基于安全单元的密钥保护。

技术规范（必须严格遵守）
目标平台：HarmonyOS 6.0 (API Level 22)，仅支持纯鸿蒙（HarmonyOS NEXT）。

语言：ArkTS（开启 strict mode，禁止 any 类型，变量必须显式声明类型，严格遵守 Sendable 协议要求）。

UI 框架：ArkUI 声明式开发范式。必须使用 Navigation + NavPathStack 路由体系（取代已废弃的 router 模块）。

安全规范：

加解密：使用 @ohos.security.cryptoFramework 实现 AES-256-GCM。

敏感数据：账号密码必须存储在 Asset Store Kit（关键资产存储服务）或经过加密的 RelationalStore 中。

身份认证：集成 @ohos.userIAM.userAuth 实现生物特征识别。

状态管理：

页面状态：@State / @Prop / @Link

跨组件：@Provide / @Consume

持久化：PersistentStorage（用于设置项）与 RelationalStore（用于结构化加密数据）。

代码风格：遵循 PascalCase 命名，组件 .ets 结尾，所有资源引用必须使用 $r()。

输出要求（分阶段、可执行）
请按以下顺序输出，每次只输出一个阶段的内容，等待我确认后再继续：

AuthManager.ets：封装 UserAuth 生物识别接口，返回认证结果。

存储工具类 (utils/storage/)：

DatabaseUtil.ets：封装 RelationalStore 的加密数据库初始化及 CRUD。

通用组件 (components/)：

PasswordStrengthBar.ets：根据密码复杂度实时变化的进度条。
